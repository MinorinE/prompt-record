# 暗房 · DARKROOM

**prompt 记录工具 — a local visual archive for images & notes**

影像与注记 · 本地档案 · no server, no upload

</div>

---

> _把每一张图与它的一句话，当作一张底片存进你自己的暗房。_

`prompt-record` 是一个**纯本地**的 WebUI 记录工具：上传一张图片，附上一段文字（prompt / 笔记 / 灵感），它就被当作一张「底片 (plate)」存入你浏览器里的档案库。没有后端、没有云端、没有上传——所有数据都留在你自己的机器上。

界面以**摄影暗房 + 编辑档案**为美学方向：暖黑底色、安全灯琥珀、胶片颗粒、取景器直角括号、自动编号 `№001 / №002 …`，以及存档时底片从「高反差深色 → 正常」的**显影动画**。

## ✨ 功能

- **三种上传方式** —— 拖拽进框 / 点击浏览 / 直接 `Ctrl+V` 粘贴截图
- **图文并存** —— 每条记录 = 一张图片 + 一段注记（可多行）
- **本地持久化** —— 使用 IndexedDB 直接存储 `Blob`，刷新、重开浏览器记录仍在
- **显影动画** —— 新存档的底片会在约 2 秒内从暗到亮「显影」浮现
- **自动编目** —— 每张底片自动编号 `№001`，并记录存档日期与时间
- **灯箱查看** —— 点击任意底片放大查看，`Esc` 关闭
- **两步删除** —— 点 `×` 先变红显示「确认?」，3 秒内再点才真删，防误触
- **快捷键** —— `Ctrl/Cmd + Enter` 在注记框内直接显影存档
- **图片预处理** —— 长边自动压缩到 1600px、JPEG 0.9，避免本地库膨胀
- **响应式** —— 桌面 / 平板 / 手机均可使用

## 🚀 快速开始

整个工具是一个**单文件**应用，无需安装任何依赖。

### 方式一：直接打开

双击 `index.html`，用浏览器打开即可。

### 方式二：本地服务器（推荐）

```bash
python -m http.server 8765
# 然后浏览器访问 http://127.0.0.1:8765/
```

> 推荐用本地服务器打开：某些浏览器在 `file://` 协议下对 IndexedDB 的处理略有差异，用 `http://` 更稳定。

## 🛠 技术栈

- 纯 **HTML / CSS / 原生 JavaScript**，零依赖、零构建
- **IndexedDB** 存储图片 `Blob`（相比 localStorage 的 base64 方案，空间翻倍且性能更好）
- **Canvas API** 做客户端图片压缩
- **URL.createObjectURL** 生成预览，配合 `revokeObjectURL` 防止内存泄漏

### 为什么用 IndexedDB 而不是 localStorage？

`localStorage` 只能存字符串、上限约 5–10MB，把图片转 base64 存进去几张就会爆。IndexedDB 可以直接存 `Blob`/`File` 对象，容量上限通常是可用磁盘空间的一大部分（Chrome 约 60%），并且读取时用 `URL.createObjectURL` 直接生成预览 URL，不经过字符串编解码，性能好得多。

## 🔒 隐私

**100% 本地。** 图片与文字都只存在你自己的浏览器里，不会发送到任何服务器。清除浏览器站点数据即等于清空档案。
